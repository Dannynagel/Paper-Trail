Model provider

Full URL of an OpenAI-compatible chat completions endpoint — local Ollama / LM Studio (leave the key blank), or Open WebUI at /api/chat/completions with Bearer auth. Privacy
Send screenshots to the model
Off (default): only the semantic action log — element labels, page titles, URLs — leaves this machine. Screenshots stay local and are spliced into the document at export. Turn on only if your screen content is cleared for the configured endpoint.
Record typed values
Off (default): field entries are logged as "Enter a value in Field" without content. Password and payment-like fields are always masked regardless of this setting.
Voice narration (transcription endpoint)
Any OpenAI-compatible /v1/audio/transcriptions endpoint — OpenAI, Groq, LocalAI, or a local Speaches / faster-whisper server (e.g. http://localhost:8000/v1/audio/transcriptions, key blank). Audio is sent here once for transcription and is never stored; only the transcript text becomes part of the recording. Some self-hosted servers accept only wav/mp3 — Chrome records webm/opus.
Must support response_format=verbose_json (segment timestamps) — whisper-1 and whisper.cpp-style servers do; OpenAI's gpt-4o-transcribe does not.

Screenshots are stored on disk (IndexedDB), so long sessions are fine; the ceiling guards against runaway recordings.
